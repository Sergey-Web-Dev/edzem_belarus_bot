# Сводка изменений для решения проблемы сна на Render

## Добавлен keep-alive механизм для предотвращения сна приложения

### Основные изменения:

1. **Новый файл `src/keepAlive.ts`**:
   - Сервис для периодического пинга приложения
   - Пинг каждые 14 минут (меньше 15 минут сна Render)
   - Health check сервер на порту 10000
   - Корректная остановка при получении сигналов завершения

2. **Обновлен `src/index.ts`**:
   - Импорт и инициализация keep-alive сервиса
   - Интеграция с существующим health check
   - Остановка keep-alive при завершении работы бота

3. **Обновлены зависимости в `package.json`**:
   - Добавлен `node-fetch` для HTTP запросов
   - Добавлен `@types/node-fetch` для TypeScript

4. **Обновлены файлы окружения**:
   - `.env` - добавлена переменная `APP_URL`
   - `.env.example` - добавлена переменная `APP_URL`

5. **Документация**:
   - `RENDER_DEPLOYMENT.md` - инструкция по деплою на Render
   - `README_KEEPALIVE.md` - описание решения проблемы

### Как работает решение:

1. При запуске приложения стартует keep-alive сервис
2. Каждые 14 минут отправляется GET запрос на `APP_URL`
3. Это поддерживает активность приложения на бесплатном тарифе Render
4. Приложение больше не засыпает после 15 минут бездействия

### Инструкция для деплоя:

1. Добавьте в Render переменную окружения:
   ```
   APP_URL=https://edzem-belarus-bot.onrender.com
   ```

2. Перезапустите сервис в Render Dashboard

3. Проверьте логи - должны появиться сообщения о пингах
