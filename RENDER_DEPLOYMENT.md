# Деплой на Render с Keep-Alive

## Что было добавлено

1. **Keep-Alive механизм**: Приложение теперь автоматически пингует себя каждые 14 минут, чтобы предотвратить сон на бесплатном тарифе Render.

2. **Новые зависимости**:
   - `node-fetch` для HTTP запросов
   - `@types/node-fetch` для TypeScript типов

3. **Новая переменная окружения**:
   - `APP_URL` - URL вашего приложения на Render (по умолчанию: `https://edzem-belarus-bot.onrender.com`)

## Как работает Keep-Alive

1. При запуске приложения стартует keep-alive сервис
2. Каждые 14 минут (меньше 15 минут сна) приложение отправляет GET запрос на свой собственный URL
3. Это поддерживает активность приложения и предотвращает переход в спящий режим
4. Сервис автоматически останавливается при получении сигналов SIGINT/SIGTERM

## Настройка на Render

### 1. Обновите переменные окружения в Render Dashboard

Добавьте переменную окружения:
- `APP_URL`: `https://edzem-belarus-bot.onrender.com` (или ваш кастомный домен)

### 2. Перезапустите сервис

После добавления переменной окружения перезапустите сервис в Render Dashboard.

### 3. Проверка работы

После деплоя проверьте логи в Render Dashboard. Вы должны видеть сообщения:
```
Starting keep-alive service. Will ping every 14 minutes.
Pinging https://edzem-belarus-bot.onrender.com at [timestamp]
Ping successful: 200
```

## Альтернативные решения

Если keep-alive механизм не решает проблему полностью, рассмотрите:

1. **UptimeRobot** (бесплатно): https://uptimerobot.com
   - Настройте мониторинг каждые 5 минут
   - Бот будет пинговать ваше приложение автоматически

2. **Kaffeine** (бесплатно): https://kaffeine.herokuapp.com
   - Специализированный сервис для предотвращения сна Heroku/Render приложений

3. **Cron-job.org** (бесплатно): https://cron-job.org
   - Настройте cron задачу для пинга каждые 14 минут

## Мониторинг

Для мониторинга состояния приложения используйте:
- Логи в Render Dashboard
- Health check endpoint: `https://edzem-belarus-bot.onrender.com` (возвращает статус 200)
- Keep-alive health check: слушает на порту 10000

## Устранение неполадок

Если приложение все еще засыпает:

1. Проверьте логи на наличие ошибок пинга
2. Убедитесь, что переменная `APP_URL` установлена правильно
3. Уменьшите интервал пинга до 10 минут (измените `pingInterval` в `src/keepAlive.ts`)
4. Рассмотрите использование внешнего сервиса мониторинга (UptimeRobot)
