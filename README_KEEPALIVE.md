# Решение проблемы сна приложения на Render

## Проблема
Приложение на бесплатном тарифе Render засыпает после 15 минут бездействия. Это приводит к тому, что Telegram бот перестает отвечать до тех пор, пока кто-то не зайдет на сайт вручную.

## Решение
Добавлен keep-alive механизм, который автоматически пингует приложение каждые 14 минут, предотвращая переход в спящий режим.

## Что было изменено

### 1. Новые файлы
- `src/keepAlive.ts` - сервис keep-alive
- `RENDER_DEPLOYMENT.md` - инструкция по деплою

### 2. Измененные файлы
- `src/index.ts` - интеграция keep-alive сервиса
- `package.json` - добавлены зависимости
- `.env` и `.env.example` - добавлена переменная `APP_URL`

### 3. Новые зависимости
- `node-fetch` - для HTTP запросов
- `@types/node-fetch` - TypeScript типы

## Как работает

1. При запуске приложения стартует keep-alive сервис
2. Сервис отправляет GET запрос на `APP_URL` каждые 14 минут
3. Это поддерживает активность приложения на Render
4. При получении сигналов завершения (SIGINT/SIGTERM) сервис корректно останавливается

## Настройка для деплоя

### 1. Добавьте переменную окружения в Render
```
APP_URL=https://edzem-belarus-bot.onrender.com
```

### 2. Перезапустите сервис в Render Dashboard

### 3. Проверьте логи
В логах должны появиться сообщения:
```
Starting keep-alive service. Will ping every 14 minutes.
Pinging https://edzem-belarus-bot.onrender.com at [timestamp]
Ping successful: 200
```

## Альтернативные варианты

Если keep-alive механизм не решает проблему полностью:

1. **UptimeRobot** - бесплатный мониторинг каждые 5 минут
2. **Kaffeine** - специализированный сервис для Heroku/Render
3. **Cron-job.org** - бесплатные cron задачи

## Устранение неполадок

1. Проверьте, что переменная `APP_URL` установлена правильно
2. Убедитесь, что в логах есть сообщения о пингах
3. При необходимости уменьшите интервал пинга в `src/keepAlive.ts`
4. Рассмотрите использование внешнего сервиса мониторинга
